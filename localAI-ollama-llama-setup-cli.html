Tổng quan & khái niệm

Local AI là xu hướng chạy các mô hình ngôn ngữ lớn (LLM) ngay trên máy tính cá nhân hoặc server của bạn, thay vì phải gọi API cloud. Hai công cụ phổ biến nhất hiện nay là Ollama và llama.cpp.

🦙 Ollama

Dành cho người dùng phổ thông
  • Giao diện CLI đơn giản, dễ dùng
  • Tự động quản lý model (pull / run / list)
  • Tích hợp sẵn REST API (mặc định port 11434)
  • Hỗ trợ GPU acceleration (NVIDIA, AMD, Intel)
  • Có one-click installer cho macOS, Windows, Linux
  • Phù hợp: cá nhân, developer, ai cũng dùng được

⚙️ llama.cpp

Dành cho kỹ thuật & tối ưu
  • C++ thuần, chạy được trên CPU, RAM thấp
  • Hỗ trợ quantization đa dạng (GGUF: q4_0, q8_0, ...)
  • Inference engine — không có CLI quản lý model
  • Cung cấp server HTTP riêng (llama-server)
  • Dùng làm backend cho nhiều tool khác (Ollama cũng dùng llama.cpp!)
  • Phù hợp: kỹ thuật, nhúng, tối ưu hiệu năng
💡 Quan hệ giữa hai công cụ

Ollama thực chất sử dụng llama.cpp làm inference engine bên trong (cùng với một số backend khác). Nói cách khác: Ollama = llama.cpp + CLI quản lý model + REST API. Nếu bạn muốn đơn giản, dùng Ollama. Nếu bạn muốn kiểm soát chi tiết, dùng llama.cpp trực tiếp.

Cài đặt Ollama

2.1. Cài bằng script (Linux / macOS)

Mở terminal và chạy lệnh duy nhất:

Bashcurl -fsSL https://ollama.com/install.sh | sh
✅ Sau khi cài
Kiểm tra: ollama --version — nếu ra số version là OK. Mặc định Ollama đã tự chạy service nền.

2.2. Cài bằng .exe (Windows)

1

Truy cập ollama.com/download

2

Nhấn nút Download for Windows — tải file OllamaSetup.exe (~300 MB).

3

Chạy file OllamaSetup.exe → Next → Install (không cần cấu hình gì thêm).

4

Sau khi cài, Ollama chạy ngầm dưới dạng Windows Service và có sẵn icon ở system tray.

5

Mở Terminal / PowerShell và kiểm tra:

PowerShellollama --version

2.3. Lệnh cơ bản sau khi cài

Bash# Liệt kê model đã tải
ollama list

# Tải model (ví dụ: Llama 3.2 3B - nhẹ)
ollama pull llama3.2:3b

# Tải model lớn hơn
ollama pull llama3.2:8b
ollama pull qwen2.5:7b
ollama pull mistral:7b

# Chạy model (chat ngay trong terminal)
ollama run llama3.2:3b

# Kiểm tra REST API (khi Ollama đang chạy)
curl http://localhost:11434/api/tags
📌 Lưu ý model

Model càng lớn càng cần nhiều RAM/VRAM. Với máy không có GPU mạnh, nên bắt đầu với model 3B-8B tham số. Ollama tự động chọn lượng tử hóa (quantization) phù hợp.
Xem danh sách model: ollama.com/library

Cài đặt llama.cpp

3.1. Cài bằng git clone + make (Linux / macOS / WSL)

1

Cài dependencies:

Bash# Ubuntu/Debian
sudo apt update
sudo apt install build-essential cmake git

# macOS (Homebrew)
brew install cmake git
2

Clone và build:

Bashgit clone https://github.com/ggerganov/llama.cpp.git
cd llama.cpp

# Build CPU-only (nhanh, ai cũng chạy được)
cmake -B build
cmake --build build --config Release -j

# Hoặc build với CUDA (nếu có GPU NVIDIA)
cmake -B build -DGGML_CUDA=ON
cmake --build build --config Release -j
3

Add vào PATH (tiện dùng mọi nơi):

Bash# Thêm vào ~/.bashrc hoặc ~/.zshrc
export PATH="$PATH:$HOME/llama.cpp/build/bin"

3.2. Cài bằng .exe / pre-built binary (Windows)

1

Vào github.com/ggerganov/llama.cpp/releases

2

Tìm bản mới nhất, kéo xuống Assets và tải file phù hợp:

  • llama-bin-win-cuda-x64.zip — nếu có GPU NVIDIA
  • llama-bin-win-avx2-x64.zip — CPU Intel/AMD đời mới
  • llama-bin-win-avx-x64.zip — CPU đời cũ hơn
3

Giải nén vào thư mục, ví dụ C:\llama.cpp

4

Thêm thư mục vào PATH (để gõ lệnh từ mọi nơi):

PowerShell# Mở PowerShell Admin và chạy:
[Environment]::SetEnvironmentVariable(
    "Path",
    [Environment]::GetEnvironmentVariable("Path", "Machine") + ";C:\llama.cpp",
    "Machine"
)

Hoặc: Settings → System → About → Advanced system settings → Environment Variables, thêm C:\llama.cpp vào Path.

3.3. Build thủ công trên Windows (CMake + Visual Studio)

Nếu muốn tự build (có GPU acceleration tốt hơn):

3.4. Download model GGUF và chạy thử

Bash# Tải model GGUF (Llama 3.2 3B - ~2 GB)
# Model GGUF có thể tìm trên Hugging Face
curl -LO https://huggingface.co/bartowski/Llama-3.2-3B-Instruct-GGUF/resolve/main/Llama-3.2-3B-Instruct-Q4_K_M.gguf

# Chạy chat
cd llama.cpp/build/bin
./llama-cli -m /path/to/Llama-3.2-3B-Instruct-Q4_K_M.gguf \
    -p "Hello, how are you?" \
    -n 128

# Chạy server HTTP
./llama-server -m /path/to/model.gguf \
    --host 0.0.0.0 \
    --port 8080

# Kiểm tra server
curl http://localhost:8080/v1/completions \
    -H "Content-Type: application/json" \
    -d '{"prompt":"Hello","max_tokens":50}'
🔍 llama-cli vs llama-server

llama-cli (cũ gọi là main) dùng để chạy inference một lần từ CLI.
llama-server (cũ gọi là server) mở HTTP server với API tương thích OpenAI, cho phép nhiều ứng dụng kết nối đến.

Tạo service chạy 24/7

Phần này hướng dẫn từng bước cụ thể để biến Ollama và llama.cpp thành service nền — tự động khởi động cùng máy, chạy 24/7 không cần can thiệp tay, tự restart nếu crash.

A. Ollama service trên Ubuntu (systemd)

✅ Tin vui

Khi cài Ollama bằng script curl -fsSL https://ollama.com/install.sh | sh, systemd service đã được tạo tự động. Bạn chỉ cần kiểm tra và cấu hình thêm. Không cần tự tạo file service.

🔹 Bước 1: Kiểm tra service đã chạy chưa

1

Mở terminal và gõ lệnh:

Bashsudo systemctl status ollama

Kết quả mong đợi — thấy 3 dấu hiệu sau là OK:

● ollama.service - Ollama Service
   Loaded: loaded (/etc/systemd/system/ollama.service; enabled; preset: enabled)
   Active: active (running) → 🟢 đang chạy

Nếu thấy Active: inactive (dead) hoặc not found → chưa cài Ollama hoặc service chưa chạy. Quay lại mục 2.1 để cài.

🔹 Bước 2: Bật tự động khởi động cùng máy

2
Bashsudo systemctl enable ollama

Giải thích: enable tạo symlink trong /etc/systemd/system/multi-user.target.wants/ — đảm bảo khi server reboot, Ollama tự chạy lại.

Output: Created symlink /etc/systemd/system/multi-user.target.wants/ollama.service → /etc/systemd/system/ollama.service

🔹 Bước 3: Restart service để áp dụng thay đổi

3
Bashsudo systemctl restart ollama

Luôn chạy restart sau mỗi lần sửa cấu hình.

🔹 Bước 4: Xem log real-time (debug lỗi)

4
Bashsudo journalctl -u ollama -f --no-hostname

Giải thích từng flag:

  • -u ollama — chỉ xem log của service ollama
  • -f — follow (log mới hiện real-time)
  • --no-hostname — bỏ tên máy cho gọn

Nhấn Ctrl + C để thoát khỏi chế độ follow.

Xem log cũ (không follow):

Bashsudo journalctl -u ollama --no-hostname -n 50
# -n 50: chỉ lấy 50 dòng cuối

🔹 Bước 5: Mở API ra ngoài mạng (nếu cần)

Mặc định Ollama chỉ chấp nhận kết nối từ localhost. Để máy khác trong LAN gọi được:

5a

Mở override config của systemd service (tạo file drop-in, không sửa file gốc):

Bashsudo systemctl edit ollama

Lệnh này mở trình soạn thảo (nano/micro). Nếu lần đầu, file sẽ trống. Hãy gõ:

Bash[Service]
Environment="OLLAMA_HOST=0.0.0.0:11434"

Giải thích: 0.0.0.0 = lắng nghe tất cả địa chỉ mạng, 11434 = port mặc định.

Lưu ý: nếu không quen nano, chạy lệnh sau để dùng echo thay thế:

Bashsudo mkdir -p /etc/systemd/system/ollama.service.d
echo -e '[Service]\nEnvironment="OLLAMA_HOST=0.0.0.0:11434"' | sudo tee /etc/systemd/system/ollama.service.d/override.conf
5b

Reload systemd và restart:

Bashsudo systemctl daemon-reload
sudo systemctl restart ollama

daemon-reload bắt systemd đọc lại tất cả file cấu hình (kể cả override).

5c

Kiểm tra cổng đã mở:

Bashsudo ss -tlnp | grep 11434

Output sẽ có 0.0.0.0:11434 thay vì 127.0.0.1:11434.

🔹 Bước 6: Thao tác quản lý hàng ngày (cheat sheet)

Bashsudo systemctl status ollama          # Xem trạng thái (running/stopped)
sudo systemctl start ollama            # Chạy service
sudo systemctl stop ollama             # Dừng service
sudo systemctl restart ollama          # Khởi động lại
sudo systemctl enable ollama           # Tự động chạy khi boot
sudo systemctl disable ollama          # Tắt tự động boot
sudo journalctl -u ollama -f           # Xem log real-time
sudo journalctl -u ollama -n 100       # Xem 100 dòng log gần nhất

4.2. Ollama service trên Windows

✅ Tin vui

File OllamaSetup.exe tự động cài Ollama làm Windows Service. Sau khi cài, Ollama chạy ngầm và có icon ở system tray (gần đồng hồ). Không cần cài NSSM hay làm gì thêm.

🔹 Bước 1: Kiểm tra service bằng services.msc (giao diện)

1

Nhấn Win + R, gõ services.msc, Enter.

Trong danh sách, tìm dòng Ollama (xếp theo chữ O).

Kiểm tra cột Status = Running, Startup Type = Automatic.

💡 Nếu không thấy → chưa cài Ollama hoặc cài không thành công.

🔹 Bước 2: Kiểm tra bằng PowerShell

2

Mở PowerShell (Admin) — nhấp phải Start → Windows Terminal (Admin) / PowerShell (Admin).

PowerShellGet-Service ollama

Output mong đợi:

Status   Name               DisplayName
------   ----               -----------
Running  ollama             Ollama

Nếu Status là Stopped:

PowerShellStart-Service ollama

🔹 Bước 3: Đảm bảo tự động khởi động cùng Windows

3
PowerShellSet-Service ollama -StartupType Automatic

3 kiểu StartupType cần biết:

  • Automatic — tự động chạy khi Windows boot (khuyên dùng)
  • Manual — chỉ chạy khi ai đó Start-Service hoặc app gọi
  • Disabled — không chạy

🔹 Bước 4: Restart / Stop service

4
PowerShellRestart-Service ollama      # Khởi động lại
Stop-Service ollama          # Dừng
Start-Service ollama         # Chạy lại

🔹 Bước 5: Cấu hình biến môi trường cho Ollama service

Muốn đổi port, thư mục lưu model, hoặc expose ra mạng LAN? Làm theo các bước sau:

5a

Mở PowerShell (Admin)

5b

Set biến môi trường ở cấp Machine (toàn hệ thống):

PowerShell# Expose ra mạng LAN (mọi máy trong mạng đều gọi được)
[Environment]::SetEnvironmentVariable("OLLAMA_HOST", "0.0.0.0", "Machine")

# Đổi thư mục chứa model (ví dụ qua ổ D cho đỡ tốn ổ C)
[Environment]::SetEnvironmentVariable("OLLAMA_MODELS", "D:\ollama_models", "Machine")

# Giữ model trong RAM lâu hơn (mặc định 5 phút)
[Environment]::SetEnvironmentVariable("OLLAMA_KEEP_ALIVE", "30m", "Machine")

# Cho phép xử lý nhiều request cùng lúc
[Environment]::SetEnvironmentVariable("OLLAMA_NUM_PARALLEL", "4", "Machine")
5c

Restart service để áp dụng:

PowerShellRestart-Service ollama
5d

Kiểm tra: mở browser trên máy khác trong LAN, gõ http://<địa-chỉ-máy>:11434 — nếu thấy Ollama is running là OK.

🔧 Tất cả biến môi trường Ollama
OLLAMA_HOST=0.0.0.0:11434      # Địa chỉ + port (mặc định 127.0.0.1:11434)
OLLAMA_MODELS=D:\models        # Thư mục chứa model (mặc định ~/.ollama/models)
OLLAMA_KEEP_ALIVE=30m          # Giữ model trong RAM 30 phút (mặc định 5m)
OLLAMA_NUM_PARALLEL=4          # Số request song song (mặc định 1)
OLLAMA_MAX_LOADED_MODELS=2     # Số model tối đa giữ trong RAM cùng lúc
OLLAMA_DEBUG=1                 # Bật log debug (dùng khi cần tra lỗi)

4.3. Tạo llama-server service trên Ubuntu (systemd)

⚠️ Khác với Ollama

llama.cpp không có cơ chế tự cài service. Bạn phải tự tạo file .service thủ công. Làm theo từng bước dưới đây.

🔹 Bước 1: Xác định đường dẫn tuyệt đối của llama-server và model

1

Gõ lệnh để lấy đường dẫn đầy đủ:

Bashwhich llama-server
# Output mẫu: /home/ubuntu/llama.cpp/build/bin/llama-server

ls -lh /home/ubuntu/models/phi-3-mini-q4.gguf
# Output mẫu: -rw-r--r-- ... 2.3G ... /home/ubuntu/models/phi-3-mini-q4.gguf

Ghi nhớ 2 đường dẫn này — sẽ dùng ở bước 2. Nếu which llama-server không ra, dùng đường dẫn tuyệt đối tới file build được.

🔹 Bước 2: Tạo file service

2
Bashsudo tee /etc/systemd/system/llamaserver.service > /dev/null << 'EOF'
[Unit]
Description=llama.cpp HTTP Server
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=ubuntu
Group=ubuntu
ExecStart=/home/ubuntu/llama.cpp/build/bin/llama-server \
    -m /home/ubuntu/models/phi-3-mini-q4.gguf \
    --host 0.0.0.0 \
    --port 8080 \
    --ctx-size 4096 \
    --threads 8
Restart=on-failure
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
EOF

Giải thích từng dòng trong file service:

After=network-online.target Chỉ chạy service sau khi mạng đã hoạt động
User=ubuntu Chạy dưới user ubuntu (không chạy root — bảo mật hơn)
ExecStart=... Đường dẫn tuyệt đối đến llama-server + argument
Restart=on-failure Tự động restart nếu crash hoặc exit code != 0
RestartSec=5 Chờ 5 giây trước khi restart (tránh restart liên tục)
StandardOutput=journal Ghi log vào journald (xem bằng journalctl)
WantedBy=multi-user.target Chạy ở runlevel đa người dùng (mặc định, không cần GUI)

⚠️ Quan trọng: thay /home/ubuntu bằng username thật của bạn (vd: /home/yourname/). Gõ whoami để biết username.

📌 1 file service mặc định chỉ chạy được 1 model

Dòng -m /home/ubuntu/models/phi-3-mini-q4.gguf load 1 model duy nhất vào RAM. Muốn đổi model thì phải sửa file service + restart.

🔹 Cách 1: Tạo nhiều file service — mỗi file 1 model, 1 port

# /etc/systemd/system/llamaserver-phi3.service
ExecStart=.../llama-server -m /home/models/phi3.gguf  --port 8080 ...

# /etc/systemd/system/llamaserver-llama3.service
ExecStart=.../llama-server -m /home/models/llama3.gguf --port 8081 ...

# Chạy tất cả cùng lúc:
sudo systemctl start llamaserver-phi3 llamaserver-llama3

🔹 Cách 2: Dùng wrapper script — 1 file service dùng được nhiều model

Tạo script /usr/local/bin/llamaserver-wrapper.sh:

#!/bin/bash
# Script này đọc model từ biến môi trường MODEL_PATH
# Nếu không set, dùng model mặc định
MODEL="${MODEL_PATH:-/home/models/phi3-mini-q4.gguf}"

exec /home/ubuntu/llama.cpp/build/bin/llama-server \
    -m "$MODEL" \
    --host 0.0.0.0 \
    --port "${PORT:-8080}" \
    --ctx-size "${CTX:-4096}" \
    "$@"
sudo chmod +x /usr/local/bin/llamaserver-wrapper.sh

File service llamaserver.service gọi script này:

[Service]
ExecStart=/usr/local/bin/llamaserver-wrapper.sh
Environment=MODEL_PATH=/home/models/llama3.gguf
Environment=PORT=8080

Muốn đổi model — chỉ cần sửa dòng Environment=MODEL_PATH=... và restart:

sudo systemctl edit llamaserver    # sửa MODEL_PATH
sudo systemctl restart llamaserver

🔹 Cách 3: Dùng Ollama (đơn giản nhất) — tự động swap model theo request, không cần restart: ollama pull phi3:mini && ollama pull llama3.2:3b

🔹 Bước 3: Phân quyền file service

3
Bashsudo chmod 644 /etc/systemd/system/llamaserver.service

Quyền 644 = chủ sở hữu đọc-ghi, mọi người khác chỉ đọc (an toàn cho file systemd).

🔹 Bước 4: Báo systemd đọc file mới

4
Bashsudo systemctl daemon-reload

Mỗi khi thêm/sửa/xoá file .service, phải chạy lệnh này để systemd nhận diện.

Kiểm tra systemd đã thấy service chưa:

Bashsudo systemctl list-units --type=service | grep llamaserver
# Hoặc dùng:
systemctl cat llamaserver   # In ra nội dung file service (xác nhận đúng)

🔹 Bước 5: Kích hoạt tự động khởi động cùng máy

5
Bashsudo systemctl enable llamaserver

Output:

Created symlink /etc/systemd/system/multi-user.target.wants/llamaserver.service → /etc/systemd/system/llamaserver.service.

Nếu thấy File exists là đã enable trước đó — không sao.

🔹 Bước 6: Khởi động service

6
Bashsudo systemctl start llamaserver

Không có output là tốt (thành công).

Nếu lỗi, xem ngay bước 7 để biết cách debug.

🔹 Bước 7: Kiểm tra trạng thái

7
Bashsudo systemctl status llamaserver

Output tốt — service đang chạy:

● llamaserver.service - llama.cpp HTTP Server
     Loaded: loaded (/etc/systemd/system/llamaserver.service; enabled; vendor preset: enabled)
     Active: active (running) since Sat 2026-07-04 10:30:00 UTC; 5s ago
   Main PID: 12345 (llama-server)
      Tasks: 15 (limit: 4915)
     Memory: 1.2G
        CPU: 3.2s
     CGroup: /system.slice/llamaserver.service
             └─12345 /home/ubuntu/llama.cpp/build/bin/llama-server ...

Output lỗi — cần debug:

● llamaserver.service - llama.cpp HTTP Server
     Loaded: loaded (/etc/systemd/system/llamaserver.service; enabled; vendor preset: enabled)
     Active: failed (Result: exit-code) since Sat 2026-07-04 10:31:00 UTC; 2s ago
    Process: 12346 ExecStart=/home/ubuntu/llama-server ... (code=exited, status=1/FAILURE)

Nếu lỗi → chạy lệnh sau để xem chi tiết:

Bashsudo journalctl -u llamaserver -n 30 --no-hostname

Các lỗi thường gặp:

  • Exec format error — file llama-server không có quyền chạy → chmod +x /path/llama-server
  • No such file or directory — sai đường dẫn model hoặc binary
  • Cannot allocate memory — không đủ RAM để load model (xem FAQ)
  • Address already in use — port 8080 đã có app khác dùng → đổi port khác

4.5. Cách thay thế: dùng Task Scheduler (không cần NSSM)

💡 Khi nào dùng Task Scheduler thay vì NSSM?

Task Scheduler có sẵn trong Windows, không cần cài thêm gì. Tuy nhiên không mạnh bằng NSSM (không tự restart khi crash, không quản lý log tốt). Dùng Task Scheduler nếu bạn không muốn cài NSSM hoặc chỉ cần chạy đơn giản.

🔹 Bước 1: Mở Task Scheduler

1

Nhấn Win + R, gõ taskschd.msc, Enter.

Hoặc Start → gõ "Task Scheduler".

🔹 Bước 2: Tạo task bằng PowerShell Admin

2
PowerShell# Mở PowerShell Admin

$action = New-ScheduledTaskAction -Execute "C:\llama.cpp\llama-server.exe" `
    -Argument "-m C:\models\Phi-3-mini-4k-instruct-q4.gguf --host 0.0.0.0 --port 8080 --ctx-size 4096"

$trigger = New-ScheduledTaskTrigger -AtStartup

$settings = New-ScheduledTaskSettingsSet `
    -AllowStartIfOnBatteries `
    -DontStopIfGoingOnBatteries `
    -StartWhenAvailable

Register-ScheduledTask -TaskName "llamaserver" `
    -Action $action `
    -Trigger $trigger `
    -Settings $settings `
    -RunLevel Highest `
    -Description "llama.cpp AI Server - chạy 24/7"

Start-ScheduledTask -TaskName "llamaserver"

Giải thích tham số:

  • -AtStartup — chạy khi Windows boot
  • -RunLevel Highest — chạy với quyền cao nhất (tránh bị block)
  • -StartWhenAvailable — nếu lỡ giờ chạy (do shutdown) thì chạy khi máy bật lại
  • AllowStartIfOnBatteries / DontStopIfGoingOnBatteries — chạy cả khi laptop dùng pin

  • Có sẵn trong Windows

  • Khó debug hơn
  • Dùng tạm nếu không muốn cài NSSM
⚠️ Lưu ý quan trọng khi chạy service
  • RAM: model 7B Q4_K_M cần ~4-6 GB RAM. Kiểm tra trước khi enable auto-start.
  • Disk: file .gguf thường 2-8 GB. Để ý dung lượng ổ cứng.
  • GPU: nếu dùng CUDA, service chiếm VRAM vĩnh viễn — không release khi không dùng.
  • Bảo mật: --host 0.0.0.0 = mở ra tất cả mạng. Nếu server có IP public, nên đặt reverse proxy (nginx) kèm xác thực hoặc ít nhất dùng --host 127.0.0.1 và chỉ mở khi cần.
  • Cold start: lần đầu service chạy, model phải load từ ổ cứng vào RAM — có thể mất 5-30 giây tuỳ model. Đây là hành vi bình thường.

FAQ & Xử lý sự cố

❓ Ollama không nhận GPU trên Windows?

Ollama tự động dùng GPU nếu có CUDA. Kiểm tra:

PowerShellollama run llama3.2:3b
# Mở Task Manager → GPU → xem "CUDA" hoặc "Compute" có hoạt động không

❓ llama.cpp build lỗi "CUDA not found"?

Giải pháp

Cài CUDA Toolkit từ NVIDIA (phiên bản 12.x). Đảm bảo nvcc có trong PATH. Hoặc bỏ flag -DGGML_CUDA=ON để build CPU-only.

❓ Lỗi "out of memory" / "Killed" khi chạy model?

Model quá lớn so với RAM. Giải pháp:

  • Dùng model nhỏ hơn (3B thay vì 8B)
  • Dùng quantization thấp hơn (Q4_K_M thay vì Q8_0)
  • Trên Ubuntu: thêm swap sudo fallocate -l 16G /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile

❓ Ollama trên Ubuntu không chạy được do port 11434 đã dùng?

Bashsudo systemctl edit ollama
# Thêm:
[Service]
Environment="OLLAMA_HOST=127.0.0.1:11435"
sudo systemctl restart ollama

❓ Làm sao để dùng OpenAI SDK gọi local model?

ollama và llama-server đều hỗ trợ API tương thích OpenAI. Chỉ cần đổi base_url:

Pythonfrom openai import OpenAI

# Thay vì mặc định https://api.openai.com/v1
client = OpenAI(
    base_url="http://localhost:11434/v1",  # Ollama
    # base_url="http://localhost:8080/v1", # llama-server
    api_key="ollama"  # không dùng nhưng cần có
)

response = client.chat.completions.create(
    model="llama3.2:3b",  # Ollama: tên model; llama-server: bất kỳ
    messages=[{"role": "user", "content": "Hello!"}]
)
print(response.choices[0].message.content)

Bash# Ollama

# llama.cpp (chỉ cần xoá file .gguf)

❓ Tăng tốc inference?

  • GPU: dùng CUDA (NVIDIA), Vulkan (AMD), hoặc Metal (Apple Silicon)
  • CPU: build với -DGGML_NATIVE=ON để tận dụng instruction set tối đa
  • Context size: giảm --ctx-size (mặc định 2048, 8192, 16384) nếu không cần context dài
  • Threads: chỉnh --threads bằng số core CPU (không phải số thread)
