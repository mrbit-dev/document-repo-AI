Sub Agent trong Claude Code

1. Khái niệm Sub Agent là gì?

Sub Agent là các tác nhân phụ được sử dụng để chia nhỏ công việc lớn thành nhiều nhiệm vụ nhỏ hơn. Thay vì một AI phải tự xử lý toàn bộ dự án, công việc được phân chia cho nhiều agent chuyên trách.

Ví dụ:

  • Agent Backend viết API
  • Agent Frontend xây dựng giao diện
  • Agent Database thiết kế schema
  • Agent QA kiểm thử
  • Agent Reviewer đánh giá chất lượng code

Mô hình này tương tự cách một nhóm lập trình viên làm việc trong công ty phần mềm.

2. Tại sao cần Sub Agent?

  • Tăng tốc xử lý dự án lớn.
  • Giảm context overload.
  • Phân chia trách nhiệm rõ ràng.
  • Dễ review chéo.
  • Tạo cảm giác AI Team thay vì AI đơn lẻ.
  • Phù hợp với codebase hàng nghìn file.
Claude Code có thể hoạt động theo hai kiểu Sub Agent hoàn toàn khác nhau. Nhiều người thường nhầm lẫn giữa chúng.

3. Hai loại Sub Agent trong Claude Code

Loại Mô tả Có Agent thật?
Role-play Sub Agent Claude đóng vai nhiều agent trong cùng một phiên chat ❌ Không
Spawned Agent Claude tạo agent riêng để thực hiện công việc ✅ Có

4. Role-play Sub Agent (Đóng vai Agent)

Đây là cách phổ biến nhất hiện nay. Claude không thực sự tạo agent mới. Nó chỉ mô phỏng nhiều agent trong một context duy nhất.

Prompt ví dụ

Bạn là trưởng nhóm kỹ thuật.

Tạo nhóm AI gồm:

- Backend Agent
- Frontend Agent
- QA Agent
- Reviewer Agent

Nhiệm vụ:
Xây dựng hệ thống đăng nhập Laravel.

Hãy hiển thị:
- Phân công công việc
- Tiến độ từng agent
- Kết quả cuối cùng

Kết quả

  • Claude sẽ giả lập nhiều agent.
  • Tất cả vẫn nằm trong cùng một context.
  • Không có process riêng.
  • Không có worker riêng.

5. Spawned Agent (Agent Thật)

Đây là tính năng Agent Teams đang được thử nghiệm. Claude Code có thể tạo agent phụ thực sự để xử lý nhiệm vụ.

Mỗi agent có thể:

  • Đọc code
  • Phân tích riêng
  • Trả kết quả cho Agent chính
  • Thực hiện nhiệm vụ độc lập
Đây mới là mô hình Multi-Agent thực sự.

6. Cách phân biệt 2 loại

Dấu hiệu Role-play Spawn Agent
Agent thật ❌ ✅
Chỉ có một context ✅ ❌
Có task delegation Giả lập Thật
Có log sub-agent ❌ ✅
Có worker riêng ❌ ✅

11. Khi nào nên dùng Role-play Agent và Spawned Agent?

Role-play Agent

Role-play Agent là việc Claude đóng vai nhiều chuyên gia hoặc nhiều thành viên trong nhóm trong cùng một phiên làm việc.

Nên dùng khi

  • Phân tích yêu cầu dự án.
  • Lên kế hoạch phát triển.
  • Thiết kế kiến trúc hệ thống.
  • Brainstorm ý tưởng.
  • Codebase nhỏ hoặc trung bình.
  • Cần kết quả nhanh.
  • Tạo demo hoặc video minh họa AI Team.

Không nên dùng khi

  • Dự án quá lớn.
  • Cần xử lý nhiều nhiệm vụ độc lập.
  • Cần phân tích hàng trăm file.
  • Cần review nhiều phần hệ thống song song.

Ưu điểm

  • Nhanh.
  • Tốn ít token hơn.
  • Dễ sử dụng.
  • Không cần cấu hình đặc biệt.
  • Phù hợp phần lớn công việc hằng ngày.

Nhược điểm

  • Thực chất chỉ có một model xử lý.
  • Không có khả năng làm việc song song thực sự.
  • Dễ bị ảnh hưởng bởi giới hạn context.
  • Các agent không thực sự độc lập.

Spawned Agent (Sub-Agent Thật)

Spawned Agent là việc hệ thống tạo ra các agent riêng biệt để xử lý từng nhóm công việc chuyên môn khác nhau.

Nên dùng khi

  • Dự án lớn.
  • Monorepo nhiều module.
  • Codebase hàng trăm hoặc hàng nghìn file.
  • Refactor hệ thống quy mô lớn.
  • Review bảo mật.
  • Kiểm thử toàn bộ dự án.
  • Phân tích kiến trúc phức tạp.

Nhóm công việc phù hợp

  • Backend Development
  • Frontend Development
  • Database Design
  • Testing & QA
  • Security Review
  • Performance Analysis
  • Documentation
  • Architecture Review

Ưu điểm

  • Chia nhỏ công việc hiệu quả.
  • Giảm tải cho context chính.
  • Có thể xử lý nhiều nhiệm vụ độc lập.
  • Dễ review chéo giữa các agent.
  • Phù hợp với dự án doanh nghiệp.

Nhược điểm

  • Tốn nhiều token hơn.
  • Thời gian thực hiện lâu hơn.
  • Khó quản lý hơn.
  • Không phải lúc nào cũng cần thiết.

Bảng lựa chọn nhanh

Tình huống Nên dùng
Viết chức năng CRUD đơn giản Role-play Agent
Tạo Landing Page Role-play Agent
Thiết kế Database Role-play Agent
Xây dựng hệ thống Laravel hoàn chỉnh Spawned Agent
Refactor dự án lớn Spawned Agent
Review bảo mật toàn bộ hệ thống Spawned Agent
Kiểm thử hàng trăm file Spawned Agent
Brainstorm ý tưởng sản phẩm Role-play Agent
Quy tắc đơn giản: Nếu một lập trình viên có thể hoàn thành công việc trong một lần ngồi làm việc, Role-play Agent thường là đủ. Nếu công việc cần nhiều chuyên môn, nhiều giai đoạn hoặc nhiều nhóm nhiệm vụ độc lập, Spawned Agent sẽ phát huy hiệu quả hơn.

12. Kết luận

Sub Agent là một trong những khái niệm quan trọng nhất khi làm việc với Claude Code.

  • Role-play Agent = mô phỏng nhiều agent.
  • Spawn Agent = tạo agent thật.
  • Role-play phù hợp demo nhanh.
  • Spawn Agent phù hợp dự án lớn.
  • Muốn kiểm tra chính xác phải xem log và luồng delegation.

Hiểu rõ sự khác nhau giữa hai loại này sẽ giúp bạn tránh nhầm lẫn khi xây dựng video, demo hoặc workflow AI Team chuyên nghiệp.
